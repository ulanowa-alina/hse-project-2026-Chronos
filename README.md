# Chronos

Chronos - десктоп-приложение для управления временем и личной продуктивностью. В его основе лежат интерактивные Канбан-доски и инструменты планирования. Проект состоит из Qt-клиента и C++ backend-сервиса. Клиент поддерживает локальное хранение данных, синхронизацию с сервером и offline-сценарии.

## Функциональные возможности приложения
 
1. Управление аккаунтом и безопасность:
* Безопасная аутентификация: Регистрация и авторизация пользователей на основе JWT-токенов с криптографическим хэшированием паролей.
* Кастомизация профиля: Возможность редактирования личных данных и управления профилем.
* Облачное хранилище аватаров: Полноценный модуль загрузки, валидации и удаления персональных изображений профиля (хранение статики на сервере).

2. Интерактивный Канбан-менеджмент:
* Персональные доски: Создание, редактирование и удаление неограниченного количества досок.
* Гибкие колонки статусов: Полное управление этапами выполнения внутри доски - создание собственных колонок и логики движения задач.
* Управление задачами : Каждая задача поддерживает детальное описание, цветовую индикацию приоритетов, установку дедлайнов и интерактивное перетаскивание между статусами (Drag-and-Drop).
* Аналитика продуктивности: Сбор метрик по выполненным задачам для детального анализа личной продуктивности.

3. Таймер фокусировки Pomodoro:
* Фокус-сессии: Встроенный таймер для работы по методу Pomodoro с визуализацией прогресса.
* История и аналитика: логирование завершенных сессий для последующего контроля продуктивности.

4. Офлайн-режим и Синхронизация:
* Offline-first pattern: Благодаря локальной базе данных SQLite, развернутой прямо на клиенте, вы можете создавать доски, менять статусы и двигать задачи вообще без доступа к интернету - интерфейс работает мгновенно и без задержек сети.
* Умная синхронизация: При восстановлении сетевого соединения клиент автоматически инициирует фоновый обмен данными с бэкендом (PostgreSQL), разрешает конфликты версий и обновляет состояние данных на всех устройствах пользователя.


## Технологический стек

### Backend

* Язык: C++17
* Сеть (сервер): Boost.Asio
* База данных: PostgreSQL(libpqxx)
* Безопасность и шифрование: OpenSSL
* Сетевой клиент (HTTP/протоколы): libcurl
* Логирование: spdlog
* Работа с данными: nlohmann/json
* Тестирование: pytest

### Frontend

* Язык: C++17
* Интерфейс: Qt6 Widgets
* Сеть (клиент): Qt6 Network
* Локальная база данных: SQLite(Qt6 Sql)


## Структура проекта

```text
.
├── backend/                  # backend-сервис и docker-окружение
│   ├── migrations/           # SQL-инициализация базы PostgreSQL
│   ├── models/               # Описания классов бд моделей
│   ├── repositories/         # Repository слой доступа к данным
│   ├── src/
│   │   ├── db/               # Пул соединений и конфиг
│   │   ├── security/         # хеширование паролей
│   │   ├── server/           # Бизнес логика
│   │   └── storage/          # работа с S3/Yandex Object Storage
|
├── frontend/                 # Клиентская часть
│   ├── include/              # заголовки UI и инфраструктуры
│   ├── src/                  # реализация UI и инфраструктуры
│   ├── local_models/         # локальные модели клиента
│   ├── local_repositories/   # работа с локальной базой SQLite
│   ├── sync/                 # синхронизация локальной и серверный баз данных
│   └── sql/local_init.sql    # SQL-инициализация базы SQLite
├── docs/                     # API контракт и схемы БД
└── CMakeLists.txt
```

## Требования


* Компилятор: С поддержкой стандарта C++17 
* CMake версии 3.20 или выше.
* Docker и Docker Compose

* Необходимые библиотеки (Qt6, Sql, Boost минимум 1.74, PostgreSQL / libpq, libpqxx, OpenSSL, nlohmann_json, libcurl, spdlog, libargon2)

## Быстрый старт 

Запуск backend:

Перейдите в директорию backend:

```bash
cd backend
./init-env.sh
docker compose up --build
```

Важно: если пересобираете `.env`, перед повторной сборкой очистите БД командой
```bash
docker compose down -v
```

### Запуск frontend

Перейдите в директорию frontend:

```bash
cd frontend
cmake -S . -B build
cmake --build build
cmake --build build --target run
```
